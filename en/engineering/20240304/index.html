As previously introduced in this blog, we wondered how the March 2024 versions of chatGPT 4 and Gemini Advanced would handle solving a physical model’s Matlab code.

↓ First, chatGPT’s approach

function pendulum_simulation
    m = 1;      % Mass
    c = 0.1;    % Damping coefficient
    k = 10;     % Spring constant
    F = 1;      % Magnitude of external force

    % Time range and initial conditions
    tspan = [0 10]; % Time range for the simulation
    x0 = [0; 0];    % Initial displacement and velocity

    % Solve the system using an ODE solver
    [t, x] = ode45(@(t, x) system_dynamics(t, x, m, c, k, F), tspan, x0);

    % Plot the results
    figure;
    plot(t, x(:,1));
    title('Displacement over Time');
    xlabel('Time (s)');
    ylabel('Displacement (m)');
end

function dxdt = system_dynamics(t, x, m, c, k, F)
    % Displacement and velocity
    x1 = x(1);
    x2 = x(2);

    % External force (constant in this example)
    % In real problems, you may need to define it as a time function, f = F(t).
    f = F;

    % Convert the second-order differential equation to a system of first-order equations
    dx1dt = x2;
    dx2dt = (1/m)*(f - c*x2 - k*x1);

    % Return the results
    dxdt = [dx1dt; dx2dt];
end

Impressive. The approach differs slightly from last time, but it’s still valid.
